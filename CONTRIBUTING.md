# Contributing

Thanks for helping make these roadmaps and project briefs better. This repo is a
practical, no-fluff guide to the skills employers hire for across seven
engineering roles, with every project mapped to a real ticket you can fix in a
live cloud workspace on [HeyDevJob](https://heydevjob.com).

## Ways to contribute

- **Improve a roadmap** - fix an inaccuracy, add a missing skill, sharpen wording.
- **Add or sharpen a project brief** - real broken code, a clear scenario, what
  you learn, what it proves.
- **Add an interview question or answer** - something employers actually ask.
- **Extend a cheatsheet** - a command or pattern you reach for on the job.
- **Fix typos, links, or formatting.**

Small fixes are welcome as direct pull requests. For anything larger (a new
roadmap stage, a batch of briefs), open an issue first so we can align.

## Project brief format

Match the existing briefs in `projects/<role>/`. Each one:

1. Opens with the **role, level, type, and skills** line.
2. Leads with the **outcome** - what the reader will be able to do and prove.
3. Shows the **real broken code** and the symptom.
4. Explains **how you'll approach it**, **what you'll learn**, and **what it
   proves** (with a resume-ready line).
5. Ends with a **"Build it for real"** pointer to the matching HeyDevJob ticket.

If your brief maps to a HeyDevJob ticket, add the row to `TICKET-MAP.md`.

## Voice rules

These keep the whole library consistent. Please follow them:

- **Lead with the outcome** (get hired, job-ready, what you can prove), not the
  activity.
- Say **"production"**, **"real broken systems"**, **"ticket"**, **"fix"**. Avoid
  "labs", "challenges", "exercises".
- **No em dashes.** Use hyphens (-) or rewrite the sentence. This is strict and
  enforced across the repo.
- Be concise, confident, and technically accurate. Cut filler.

## Submitting a change

1. Fork the repo and create a branch (`fix/devops-roadmap-typo`).
2. Make your change, keeping the voice rules above.
3. Open a pull request describing what changed and why.
